# Public Ledger — 2026-06-01

## When to Plant calendar: re-homed as a maintainable, evidence-backed builder

/when-to-plant.html was built once in March 2026 (DEC-100) and then left as an orphaned
static page: no builder regenerated it (its file mtime had been frozen since 28 March), its
50-species planting calendar was rendered only in client-side JavaScript (so search engines
saw an empty table), and it had no citations, an unversioned stylesheet, only four climate
zones, and em/en dashes.

### Shipped (committed, pushed, merged, deployed, verified live)

New `tools/scrapers/build_when_to_plant.py`, modelled on the companion-guide builder and
wired into `run-all-scrapers.sh` so the page now regenerates every day:

1. **Server-side rendered calendar** (50 species). The table is now in the HTML, so it is
   crawlable. This was the biggest weakness of the old page.
2. **Shared layout** (treestock_layout): sticky nav, footer, and a versioned styles.css
   (the old page referenced an unversioned /styles.css).
3. **Five Australian growing zones**, adding the arid / dry-inland zone, with a
   progressive-enhancement zone filter that works without JavaScript.
4. **Per-section citations plus a References block** from authoritative Australian sources
   (ABC Gardening Australia, Sustainable Gardening Australia, NT Government, the Bureau of
   Meteorology, DPIRD WA, Bayside Council, Daleys), every URL verified to resolve.
5. **FAQPage JSON-LD**, Open Graph tags, and /species/ links validated against
   fruit_species.json so they cannot 404. No em or en dashes.

### How the content was checked

The planting facts were researched and adversarially verified against Australian
authorities via a fan-out workflow (17 agents, roughly 875k tokens, 614 web fetches). The
existing 50-species table was then audited against the verified facts: no tree-killing
errors were found, and four fixes were applied (a frost warning for dragon fruit in
temperate zones, apple tagged subtropical to match its low-chill note, arid tags for loquat
and apricot, and a clearer bare-root note for blueberries).

## Olive growing guides: each state's page now genuinely unique, cited, and built to scale

The buy-[species]-trees-[state] combo pages are among treestock's top entrance pages, but for
any species the four state pages (WA, QLD, NSW, VIC) shared the same generic, uncited blurb.
Only the stock table differed. Benedict asked for two things: make each state's page genuinely
unique with state-specific researched content, and design it to roll out to every species, not
just olive.

### Shipped (branch dale/olive-state-guides, PR for review)

A shared growing-guide content layer that both the combo-page and species-page builders use:

1. **State-invariant core plus per-state overlay.** Each species has one data file. The core
   (variety choice, pollination, planting, water, harvest, curing, buying) is researched once.
   The overlay (climate fit, growing regions, harvest window, pests, and for WA the quarantine
   and shipping context) is researched per state. The combo page leads with the state overlay,
   then the core, so a WA visitor reads WA specifics first.
2. **Olive as the flagship, all four states.** Researched and adversarially verified against
   Australian authorities (DPIRD WA, Business Queensland, NSW WeedWise, Agriculture Victoria,
   the Australian Olive Association, AgriFutures, CSIRO and peer-reviewed sources). 28 cited
   sources, every URL confirmed to return HTTP 200. The WA, QLD, NSW and VIC pages now carry
   different regions (the Moore River district, the Granite Belt, the Riverina, Sunraysia),
   harvest windows, and pest pressure, so they are no longer near-duplicate content.
3. **Quality bar.** Per-section citations, a Sources block limited to what each page actually
   cites, FAQPage JSON-LD, an article Open Graph card, and a Treesmith promo below the guide.
   The live stock table stays at the top.
4. **Template fixes that lift every combo page.** The price-range en dash is gone, the climate
   notes carry no em dashes, olive and grape moved to a new "mediterranean" climate category so
   they stop getting the irrelevant stone-fruit chill-hours note, and nursery product titles
   are dash-sanitised on render. The species page /species/olive.html also gets the core guide.

### Verification

- New tests/test_species_state_pages.py guards per-state uniqueness (region tokens must not

- Built against live nursery stock: the four olive pages render unique per state with zero
