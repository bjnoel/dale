# 2026-07-23 — Funnel audit finds primary Treesmith leak

## What happened

Two tickets completed in today's session.

**DAL-217: Fixed golden test non-determinism**

The test suite was failing on dale-server (but not locally) for two golden tests:
`test_bush_tucker_landing` and `test_rare_finds`. Root causes:

1. `RARITY_SCORES_FILE` was hardcoded to `/opt/dale/data/rarity_scores.json` in
   both `build-dashboard.py` and `build_rare_finds.py`. On the server, this file
   exists with live computed data. Tests should use the committed fixture at
   `tests/golden/fixture/rarity_scores.json`. Fixed by resolving the path relative
   to `data_dir.parent` at runtime.

2. `hard_to_find_slugs` was a Python `set`; `list(set)` has non-deterministic
   ordering. Fixed with `sorted()`. Full suite: 1,799 tests green.

**DAL-218: Treesmith funnel audit**

Pulled data from Plausible, PostHog, and Resend to find where the
treestock → Treesmith → Pro funnel leaks.

Primary finding: only 12 unique visitors reached /treesmith.html in 30 days,
out of 2,865 treestock visitors (0.4% CTR). The homepage (225 visitors/month,
our top page) has no Treesmith CTA. The CTAs exist on species/variety/guide pages
but not on the homepage, nursery pages, or compare pages.

The Treesmith app itself has 43 monthly active users and 6 new installs last week.
Post-install: 33% add a plant (good), 0% log a subsequent activity (bad — the key
differentiator is not being used). One paywall view in 7 days, zero Pro purchases.

What was shipped:
- Added UTM parameters to all Treesmith web CTAs so Plausible can attribute clicks
  by page type
- Added UTM to the iOS App Store link so Apple Analytics can track referrals
- Emailed Benedict the full audit findings

Proposed for Benedict's approval:

  existing species-page promo block) — this is the highest-leverage single change
