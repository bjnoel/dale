# Public Ledger, 2026-06-08

## One variety, one page: cleaning up how treestock groups cultivars

treestock builds a page per fruit variety ("where can I buy a Hass avocado in Australia"),
and those pages were splitting apart whenever nurseries wrote the same variety in different
ways. A black sapote listed as "Mossman 5l" at one nursery, "Mossman (grafted)" at another,
and plain "Mossman" at a third became three separate pages instead of one. Dorsett Golden
apple split four ways. Pot sizes, "(PICK UP ONLY)", "QLD ONLY", "Tubestock", and even a
"bear rooted" typo were all leaking into the page names.

The fix teaches treestock to read past that noise. Before it decides which variety a listing
belongs to, it now strips pot sizes (2L, 90mm, 20cm, 140ml), shipping notes (QLD only,
restricted to SE QLD, pickup only), propagation labels (grafted, bare rooted, cutting grown,
tubestock), and size words (dwarf, super dwarf, standard, large, medium, small). Then it
groups every nursery's version of that variety onto a single page.

The result: the variety section went from about 5,266 pages to 3,971, because 1,295
duplicates collapsed into the right place. Blue Java banana now gathers four nurseries on one
page instead of scattering across several. Pages also got cleaner titles ("Black Sapote -
Mossman" rather than "Black Sapote Mossman 5l").

### Keeping real distinctions

Not every size word is noise. In bananas, "Dwarf Cavendish" and "Dwarf Ducasse" are genuine,
distinct varieties, not just a small Cavendish, so the cleanup leaves bananas' "Dwarf" alone.
"Mammoth", "Giant", and "Jumbo" stay too, because they are part of real cultivar names (Feijoa
Mammoth, for one). And "QLD" on its own is kept where it marks a real botanical form, like the
Queensland Davidson plum.

### Restock alerts kept working

A handful of people had set restock alerts on the old page names. Those were carried across to
the new names (an alert on "Sapodilla Grafted Krasuey" now watches "Sapodilla Krasuey"), so
nobody loses an alert they signed up for. The alert database was backed up first.

### How it was built

Written test-first: the messy real-world examples went in as failing tests, then the cleanup
code was written until they passed, with the full test suite (1,387 tests) green before
anything shipped.

## Bringing the detail back: a type label on each nursery row

Grouping every version of a variety onto one page was the right call, but it had a side
effect. Once "Apple Gala Dwarf", "Apple Gala (Bare Rooted)" and plain "Apple Gala" all land
on the same page, the rows look identical even though the trees are not. A shopper comparing
prices could not see that one nursery sells a dwarf, another a bare-rooted tree, and a third
both a standard and a dwarf.

So each nursery row now carries a small label when the tree is not a plain standard one:
Super Dwarf, Semi Dwarf, Dwarf, Bare rooted, Tubestock, Cutting grown, or Advanced. Where a
listing is more than one of these, they combine (for example "Super Dwarf, Bare rooted").
Grafted and standard trees, and plain pot sizes, get no label, because that is the default
and a label there would just add clutter.

The label reads from the exact same word list the grouping code strips out, so the two can
never disagree: anything removed from the page name to make varieties group is exactly what
shows up on the row. And if the variety name already says it (a banana literally called
"Dwarf Cavendish"), the redundant label is hidden.

On the Gala apple page this now reads cleanly: Yalca shows Dwarf, Aus Nurseries shows Bare
rooted, and Guildford shows one standard row alongside a dwarf one. Shipped test-first with
the full suite (1,389 tests) green.
