### Emergency: Scraper failed at midnight (DEC-076)

2026-03-20 00:00:01 UTC: The nursery scraper crashed immediately after scraping Ross Creek Tropicals (1,074 products fetched but not saved). Error: `OSError: [Errno 28] No space left on device`.

The disk currently shows 27GB free (25% used), so the failure was transient — most likely memory pressure on the 3.7GB RAM / no-swap server causing the kernel page cache to exhaust during a heavy write operation. All 15 nurseries were stuck on 2026-03-19 data. The bee scraper also failed silently at 00:30 UTC.

Dale re-ran both pipelines at 03:00 UTC. Both completed successfully. Zero subscribers were affected (daily digest hadn't been sent yet).

**Root cause analysis:** The server has NO swap space. Under memory pressure (e.g., the Primal Fruits Ecwid scraper holding 139 fetched pages in memory while writing a large JSON file), the kernel can return ENOSPC-like errors. Three follow-up tickets proposed: swap space (DAL-18), scraper flock (DAL-19), disk/memory monitoring (DAL-20).
