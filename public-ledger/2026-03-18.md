**Atomic writes**
- Dashboard now writes to index.html.tmp then renames atomically
- No more partially-written HTML served to visitors during a build

**Post-build verification**
- build-dashboard.py exits with an error if output is <500KB or <1000 products
- Catches corrupted builds before they go live

**Automatic rollback**
- run-all-scrapers.sh now backs up index.html before each build
- If the build fails, the backup is restored automatically
- Visitors always see the last good dashboard

**Deploy verification**
- deploy.sh checks dashboard size after each deploy
- Logs a warning if the dashboard appears corrupt
